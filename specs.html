airgap // qr transfer · spec sheet

Move files between offline devices
with just a camera.

Fountain-coded file transfer over a stream of QR codes. No network, no cable, no Bluetooth. Missed frames don't matter — the receiver finishes whenever it has enough.

Transfer protocol

Coding schemeLT fountain code (Luby, 2002)
Frame-loss tolerance50%+ — decoder finishes on any sufficient subset
Decode overhead~1.10–1.17× the number of source blocks
PRNGmulberry32, seeded per symbol
Degree distributionIdeal-soliton-ish, 6% degree-1 spike
Integrity checkFNV-1a 32-bit hash in the header
EndpointSender emits indefinitely; receiver self-stops when decoded

Maximum file size

There are two separate ceilings: what the device can hold, and what your patience allows.

Technical ceiling — device memory

The receiver reconstructs the whole file in RAM several times. No hard cap exists in the code; the chunk-count field alone allows 4.3 billion blocks.

Known limitations

No encryptionAnyone who sees the screen can capture the data
No two-way ACKSender can't tell what the receiver has; it just keeps emitting
Memory-boundWhole file reconstructed in RAM — large files strain mobile devices
Hash not enforcedIntegrity hash is carried but not verified on completion
Single-cameraReceiver needs a rear camera; sender needs a bright screen
